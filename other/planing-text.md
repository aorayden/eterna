# План разработки проекта «Eterna»

### 1. Название проекта
**Eterna — Галерея произведений искусств (Eterna - Art Gallery)**

### 2. Описание проекта
Eterna — это кроссплатформенное мобильное приложение для коллекционеров и любителей искусства, разработанное на фреймворке Flutter. Приложение предоставляет удобный интерфейс для каталогизации произведений с возможностью просмотра детальной информации, «умной» фильтрации по историческим эпохам и поиска по названию. Ключевыми особенностями являются полная автономность (офлайн-режим с локальным хранением данных), а также поддержка импорта и экспорта базы данных в формате JSON для обмена коллекциями.

### 3. Структура данных (JSON)
Каждое произведение искусства описывается следующим объектом:

```json
{
  "title": "String (Название произведения)",
  "description": "String (Текстовое описание)",
  "yearStart": "String (Год начала создания, формат DD.MM.YYYY или YYYY)",
  "yearEnd": "String (Год окончания создания, формат DD.MM.YYYY или YYYY)",
  "genre": "String (Жанр: painting, sculpture, photo, graphics)",
  "imageUrl": "String (URL-ссылка на обложку произведения)"
}
```

### 4. Распределение обязанностей

**Тимур (UI/UX Дизайн):**
*   Разработка концепции приложения.
*   Создание дизайн-системы в Figma:
    *   Определение глобальных стилей (цветовая палитра, типографика).
    *   Разработка библиотеки компонентов (атомарный дизайн: кнопки, инпуты, карточки).
*   Прототипирование всех экранов приложения (Галерея, Импорт, Экспорт, Детальный просмотр).

**Ильнур (Full-stack разработка):**
*   Настройка архитектуры проекта на Flutter.
*   Верстка экранов (Pixel-perfect) на основе макетов Figma.
*   Реализация кастомного UI Kit (виджеты).
*   Разработка бизнес-логики:
    *   Сервис хранения данных (`ArtworkService`).
    *   Логика фильтрации и поиска.
    *   Реализация импорта (парсинг JSON) и экспорта (генерация файла + `share_plus`).
